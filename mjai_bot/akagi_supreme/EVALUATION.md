## 修正内容まとめ

### 1. game_state.py — 脅威度推定の大幅強化
- **手出し/ツモ切りパターン追跡**: ツモ切り連続後の手出し = テンパイ気配を検出
- **河の染め偏り分析**: ホンイツ/チンイツ/トイトイ気配を副露＋河から検出
- **受入枚数推定**: 手牌構成から有効牌の枚数を概算
- **巡目対応脅威度**: `apparent_threat_level(current_turn)` で巡目を反映
- **赤ドラ二重カウント修正**: `tile_base()` を使い正規化後にドラ照合
- **点差逆転計算**: `points_needed_for_placement()`, `min_han_for_points()` 追加
- **自風/場風/役牌判定**: `is_my_yakuhai()` 追加

### 2. push_fold.py — 巡目ベースの攻守判断
- **巡目 (`my_turn`) を主軸に**: `remaining_tiles` だけでなくプレイヤー巡目で分岐
- **テンパイ閾値厳格化**: トップ勢と同じくテンパイからほぼ降りない（safety_weight極小化）
- **イーシャンテンの適正化**: 序盤/中盤/終盤で異なる safety_weight（0.03〜0.18）
- **打点評価の精密化**: 役牌(自風/場風含む)、ホンイツ、メンゼン立直＋裏ドラ期待を反映
- **打点→翻数変換**: 正しい点数表ベースの翻数→点数マッピング
- **脅威度に行動パターン反映**: `tedashi_after_tsumogiri_streak()` によるダマ聴検出

### 3. placement_strategy.py — 精密なオーラス戦略
- **逆転条件計算**: `points_needed_for_placement()` + `min_han_for_points()` でツモ/ロン別に必要翻数を算出
- **ダマテンの適用範囲拡大**: オーラス1位だけでなく、南場1位・跳満以上の手でも適用
- **供託棒の戦略反映**: kyotaku_bonus を all-last, south の両方で参照
- **親番4位連荘戦略**: 副露推奨 (meld_multiplier=1.2) で速攻アガリ優先
- **2位/3位の精密分岐**: 1位との翻数差、3位との点差を同時考慮

### 4. strategy_engine.py — バグ修正と判断改善
- **HORA 見逃しバグ修正**: `mask[IDX_HORA]` チェックを `mortal_action` チェックより前に移動
- **デッドパラメータ削除**: `_find_best_discard()` から未使用の `safety_weight` 引数を除去
- **Q値正規化改善**: min-max → パーセンタイルベース (10th/90th) で外れ値に耐性
- **danger 正規化改善**: リーチ有無で max_danger を 1.8/1.2 に動的切替
- **副露判断の強化**: FOLD 時のポン判断、CAUTIOUS 時のチー判断にQ値差分を考慮
- **リーチ判断の閾値修正**: 0.85 → 0.90 でMortalのリーチ判断をより尊重

## 第二回評価（2026-03-22）: トッププレイヤーの思考反映度 — C+

全ファイル（約2,165行）を精読した上での厳密な評価。
設計思想「Mortalを最大限信頼し、順位戦略でのみ介入」は正しいが、介入ロジックの麻雀的精度に多数の問題がある。

---

### 1. 致命的問題

#### 1-1. アガリの無条件強制（strategy_engine.py:103）
```python
if IDX_HORA < len(mask) and mask[IDX_HORA]:
    return IDX_HORA
```
mask に hora がある場合は無条件でアガるが、トッププレイヤーの思考と矛盾する場面がある：
- オーラス1位でアガると着順が下がるケース（アガリ点が低すぎて他家のリーチ棒回収で2着落ち等）
- ダブロンの片アガリ選択が必要な場面

#### 1-2. `mortal_action == IDX_NONE` で即リターン（strategy_engine.py:111-112）
Mortal がパスを選択した場合、戦略エンジンは一切介入しない。
しかし、Mortal が鳴きチャンスをスルーしている場面で、順位状況的には鳴くべきケースがある
（オーラス4位で速度最優先など）。パスの上書きが一切できないのは設計上の大きな穴。

#### 1-3. 点数計算が30符固定（push_fold.py, game_state.py）
`estimate_hand_value` と `min_han_for_points` が30符基準でハードコード。
ピンフツモ（20符）、七対子（25符）、副露手の40符/50符など、符計算の差は打点に大きく影響。
特にオーラスの逆転条件計算で「必要翻数」が1翻ずれると戦略が根本から変わる。

---

### 2. 重大な問題

#### 2-1. 受入枚数が未使用
`game_state.py` に `estimate_acceptance_count()` が実装済みだが、`push_fold.py` から一度も呼ばれていない。
トッププレイヤーは「テンパイかどうか」だけでなく「何面待ちか」「有効牌が何枚残っているか」で攻守を決める。
良形テンパイ（両面8枚）と悪形テンパイ（ペンチャン4枚）で同じ判断は明らかに誤り。

#### 2-2. ダマテン判定の不完全さ（placement_strategy.py:323-384）
- 待ちの良さを考慮していない（悪形ダマは出アガリ率が低い）
- 巡目を考慮していない（序盤ダマと終盤ダマでは意味が違う）
- 他家の捨て牌から待ち牌が出やすいかの判断がない
- 「跳満以上なら常にダマ」（L379-382）は過度に単純化。跳満でもリーチで三倍満になれば逆転できるオーラス4位のケースを無視

#### 2-3. 副露判断が Q値差分のみ
鳴き判断が「Mortal の Q値」と「none の Q値」の比較のみ。トッププレイヤーが考慮する：
- 鳴いた後の手牌構成（守備力の変化）
- 鳴きによる打点変化（メンゼン維持のリーチ+裏ドラ vs 副露の速度）
- 他家への情報開示

これらが一切なく、Q値の閾値調整だけでは副露の質を上げられない。

#### 2-4. 安全牌選択の問題
- 現物（完全安全牌）の最優先が明示的に保証されていない
- `safety_weight` 上限0.80で、FOLD時でもQ値が20%反映される（ベタオリ時は安全度100%にすべき）
- ワンチャンス評価が弱すぎる（+0.08）

#### 2-5. 脅威度推定の限界
- 副露の打点推定がない（白ポン+中ポンと白ポンのみで同じ扱い）
- 手出し牌の内容を見ていない（中張牌の手出しと端牌の手出しでは意味が違う）
- 巡目ごとの門前テンパイ確率を考慮していない

---

### 3. 中程度の問題

#### 3-1. オーラス戦略の粒度不足
- 着順落ちリスクの計算が不十分
- 直撃条件の計算がない（「下家から直撃ロンすれば逆転」は頻出パターン）
- ノーテン罰符の考慮がない

#### 3-2. 染め手検出の粗さ（game_state.py:138-161）
- 門前ホンイツ/チンイツ（リーチホンイツなど）を全く検出できない
- 河の偏りが50%以上という閾値が甘い

#### 3-3. 東風戦非対応
`is_all_last` が「南4局」固定（game_state.py:342）。東風戦では「東4局」がオーラス。

#### 3-4. Q値比較のバグ（strategy_engine.py:153-157）
```python
if discard_q >= riichi_q * 0.90:
```
Q値が負の場合にこの比較は意味をなさない（負×0.9 は元より大きくなる）。差分ベースに変更すべき。

#### 3-5. 3人麻雀への非対応
GameState は4人固定。mortal3p と組み合わせた akagi_supreme3p が存在しない。

---

### 4. 良い点

- 設計思想「Mortal信頼 + 順位戦略補完」は正しい
- SupremeEngine のラッパーパターンがクリーンで既存インターフェースを壊さない
- 手出し/ツモ切り追跡が実装されており方向性は正しい
- オーラスの逆転条件計算（翻数別ロン/ツモ）の基本構造がある
- EVALUATION.md に修正履歴があり自己改善の意識がある

---

### 5. 結論

akagi_supreme は正しいアーキテクチャを持つが、介入ロジックの麻雀的精度が不足。現状は：
- **介入なし**の場面では Mortal と同等（=問題なし）
- **介入あり**の場面では不正確な判断で Mortal より悪くなる可能性がある

トッププレイヤーの思考を正しく反映するには、上記の致命的・重大な問題の修正が必須。
特に「受入枚数の活用」「符計算の導入」「アガリ/パスの例外処理」「待ち形評価」が最優先課題。

## 第二回評価に基づく修正（2026-03-22）

### game_state.py
- **東風戦対応**: `_is_tonpu` フィールド追加、`is_all_last`/`is_late_game` を東風/半荘両対応
- **符計算対応**: `_calculate_points(han, fu, is_dealer, is_tsumo)` を追加、`min_han_for_points` に `fu` パラメータ追加。20符(ピンフツモ)、25符(七対子)、30/40/50符すべて対応
- **脅威度推定強化**: `apparent_threat_level` に `round_wind`/`seat_wind` パラメータ追加
  - 風牌の役牌ポン検出（自風・場風）
  - 小三元ポテンシャル検出（三元牌ポン2つ）
  - 門前ホンイツを河から検出（`detect_honitsu_from_river()`）
  - 手出し内容分析（中張牌の手出し=強い手のシグナル）
- **直撃条件計算**: `points_needed_direct_hit()` 追加
- **ノーテン罰符**: `noten_penalty_effect()` 追加
- **`_opponent_wind()`/`_threat_of()`**: 風コンテキストを自動的に渡すヘルパー

### push_fold.py
- **受入枚数統合**: `evaluate_push_fold` に `acceptance_count` パラメータ追加
- **良形/悪形テンパイ区別**: `good_shape`（8枚以上）/`bad_shape`（4枚以下）で分岐
  - 悪形テンパイ×安手×極端な脅威 → わずかに safety_weight 付与
  - イーシャンテンでも良形なら safety_weight を引き下げ
  - リャンシャンテンの中盤で良形＋高打点 → より攻撃的に

### placement_strategy.py
- **ダマテン判定強化**: `should_damaten` に `acceptance_count` 追加
  - 悪形ダマ（出アガリ率低い）→ リーチ優先（裏ドラ期待＋威嚇）
  - 終盤（14巡目以降）→ ダマのメリット薄い（残り巡目が少ない）
  - 跳満以上でも悪形なら安易にダマにしない
- **直撃条件**: オーラス2位で1位への直撃ロンが安い翻数で可能な場合の分岐追加
- **ノーテン罰符リスク**: オーラス1位で差が3000点以内 → テンパイ維持を重視

### strategy_engine.py
- **Q値比較バグ修正**: リーチ判定の比率ベース(`riichi_q * 0.90`)を差分ベース(`q_diff`)に変更。負のQ値で正しく動作
- **副露判断も差分ベースに**: チー/ポンの Q値比較を差分ベースに統一
- **現物（現物）優先**: `_find_genbutsu_discard()` 追加。FOLD/CAUTIOUS時にリーチ者の河にある牌を最優先
  - FOLD + safety_weight >= 0.50 → 現物があれば無条件で現物
  - CAUTIOUS → Q値差が小さければ現物
- **パス上書き**: `_check_pass_override()` 追加。Mortal がパスした副露を順位状況で上書き
  - オーラス4位 + meld_multiplier >= 1.1 → Q値差0.15以内なら副露
  - オーラス3位 + 4位が近い → Q値差0.08以内なら副露
- **安全上限引き上げ**: FOLD時の `safety_weight` 上限を 0.80 → 1.0 に。ベタオリ時は安全度100%を許容
- **受入枚数の配線**: `estimate_acceptance_count()` を呼び出し `evaluate_push_fold` と `should_damaten` に渡す

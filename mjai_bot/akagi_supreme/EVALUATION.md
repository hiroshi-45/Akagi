### 1. 発見された問題

#### 1-1. 3人麻雀検出がラウンドごとに再実行され、0点プレイヤーで誤検出（game_state.py:908）

**問題**: `_handle_start_kyoku`で毎ラウンド `scores[3] == 0 and all(s > 0 for s in scores[:3])` による3P判定を行っていた。これにより：
- 3P対局中にプレイヤーが0点になると、`all(s > 0 ...)` が False → 4Pに誤判定 → 壁枚数70（正しくは55）、4人分のアクティブプレイヤーで処理
- 4P対局中にプレイヤー3が0点になると `scores[3] == 0` → 3Pに誤判定 → 壁枚数55、3人分で処理

**修正**: 初回ラウンド（`_round_count == 0`）のみ3P判定を実行し、以降は値を保持。`reset_game()`で`num_players = 4`にリセット。

**深刻度**: 中程度（3P対局で中盤以降に壁枚数や安全牌計算が狂う可能性）

#### 1-2. estimate_hand_value の親基底値が3900点で過大評価（push_fold.py:59-62）

**問題**: `estimate_hand_value`の基底値（han_estimate < 2のフォールバック）が:
- 子: 2000（≈2翻30符相当、妥当）
- 親: 3900（≈子3翻30符相当 — 親2翻30符は2900なので約1000点の過大評価）

コメントも「dealer 1 han 30 fu」と記載されていたが、1翻30符親は1500点であり完全に不正確。

**修正**: 親の基底値を2900に修正（2翻30符親ロン）。コメントも「strategic floor」として正確に記載。

**深刻度**: 軽微（han_estimate < 2のケースは稀で、判断への影響は限定的）

### 5. 主要な強み

1. **Mortalとの統合が的確**: PUSH時はMortalを100%信頼し、介入は明確なパターン検出に限定。safety_weight等の二重カウントなし
2. **オーラス戦略の精密さ**: 直撃条件、ノーテン罰符、リーチ棒コスト考慮など、トッププレイヤーの思考を正しく反映
3. **ダマテン判断の網羅性**: 山残り枚数、巡目、追っかけリーチ、順位条件を総合的に判断
4. **脅威度推定の多角性**: リーチ巡目、副露内容、手出しパターン、河の偏り、ドラポンなど多面的に評価

### 6. 改善余地

1. **estimate_hand_valueと_calculate_pointsの統合**: 前者は簡易テーブル、後者は精密計算。前者も符を考慮すると精度向上
2. **3P初期検出の脆弱性**: 初回ラウンドで正しくない可能性（例: 特殊な開始配点）。`set_num_players()`的なAPIの追加も検討余地あり
3. **double riichi時のリスク計算**: 個々のリーチ者の早いリーチ/親リーチボーナスが1.8倍の一括処理に埋もれる

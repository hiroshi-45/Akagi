## 評価基準：雀魂トッププレイヤー（魂天・雀聖上位）の思考

### 1. 攻守判断（押し引き）

- **トッププレイヤーの原則**: 「押すか降りるかは二択。中間はない」
- PUSH/MAWASHI/FOLD の三値判断が正しく機能しているか
- テンパイからはほぼ降りない（ただし安手×悪形×高脅威は例外）
- リーチ宣言済みなら無条件 PUSH
- Mortal の Q値に safety_weight を加重平均して安全度を二重カウントしていないか確認
- 巡目（`my_turn` = per-player巡目）を正しく使用しているか（全体巡目 `turn` との混同禁止）

### 2. ベタオリ（完全ベタオリ）

- **FOLD 時は安全度 100%** で牌を選択すること
- 安全牌の優先順位: **現物** → **後付け現物**（リーチ後に他家が通した牌）→ **スジ** → **壁** → **ワンチャンス**
- 現物が複数ある場合: 手牌進行に不要な（孤立した）牌を優先切り
- 赤牌の正規化: `tile_base()` で 5mr → 5m に正規化してからルックアップ

### 3. 回し打ち（MAWASHI）

- 安全牌の中から Q値でタイブレーク（回し打ちの概念）
- 牌を安全度で3段階（SAFE/MODERATE/DANGEROUS）に分類し、最安全カテゴリ内で Q値最高を選択
- MAWASHI は「中途半端に安全な牌を切る」のではなく「安全な牌の中で手牌進行に最も有利な牌を選ぶ」

### 4. 副露判断

- Q値差tendedのみでの判断は不十分だが、Mortal の Q値が既に手牌構成・守備力を内包しているため外部補正は二重カウントのリスクあり
- **役牌ポン・ドラポン**:

Mortal が選んだなら尊重（自動受理ロジックが正しく機能しているか）
- **門前-維持価verenigde**: メンゼンかつテンパイ/イーシャンテン近辺なら鳴きに Q値ペナルティをateは加味する
- **オーラス4位**: 速度最優先で門前ペナルティ撤廃。ただし「条件を無視した鳴き」（必要打点に届かない副露）は禁止
- パス上書き (`_check_pass_override`): 順位条件が明確に要求する場合のみMortal のパスを上書き

### 5. アガリ判断

- **Mortal が `IDX_HORA` を選ばなかった場合はMortal を信頼する**（アガリ見逃しの尊重）
- アガリの無条件強制（`mask[IDX_HORA]` で即 return）は禁止
- オーラスの着順条件でアガリ見逃しが正解のケースが存在する

### 6. リーチ/ダマテン判断

- ダマテン条件: 順位条件（`prefer_damaten`）、打点、待ち形、巡目、山残り枚数を総合判断
- 1種1枚の極薄待ち → ダマテン推奨（無駄な1000点出費回避）
- 山残り3枚以上 → ダマテン有利（ツモ期待あり）
- 悪形×安手 → リーチの威嚇価値で判断（状況依存）
- Q値が負の場合のリーチ乗算バグがないか確認（負×1.1 = より小さい値 → 逆効果）
  - 正しい処理: 負なら除算（`riichi_q / multiplier`）

### 7. 順位戦略（オーラス）

- **オーラス1位**: 大差リード(≥12000) → FOLD、中差(4000-12000) → MAWASHI、薄差(<4000) → PUSH維持
- **オーラス2位**: 1位との差が逆転可能か計算。直撃条件も考慮
- **オーラス3位**: 2位・4位両方との距離感で判断
- **オーラス4位**: 降りても4位は変わらない → 原則PUSH。ただし条件を無視した副露は禁止
- ノーテン罰符の考慮: 薄差1位でテンパイ維持の重要性

### 8. 順位戦略（南場・東場）

- 南場: 点差に応じた微調整。大きくビハインドなら攻撃優先
- 東場: 原則Mortal に委譲。親番はやや攻撃的（riichi/meld multiplier）

### 9. 脅威度推定

- リーチ巡目: `riichi_turn` は **per-player巡目**（`turn // 4`）で格納されているか
- 早いリーチ（≤6巡目）: 良形・高打点の可能性が高い → 脅威度+0.5
- ツモ切り連続→手出し: ダマテンパイシグナル → 正しく検出されているか（`_tsumogiri_streak_tedashi` フラグ）
- 副露手の打点推定: 白ポンのみ vs 白ポン+中ポン+ホンイツ で脅威度を区別
- 門前ホンイツの河からの検出

### 10. 点数計算

- 符計算: 20符（ピンフツモ）、25符（七対子）、30/40/50符 に対応しているか
- 三倍満親の点数: 36000（32000 は誤り）
- `_calculate_points` と `estimate_hand_value` の整合性

### 11. 受入枚数

- `_estimate_deficiency` の精度: 面子抽出で順子優先/刻子優先の両方を試行しているか
- 3人麻雀: 2m〜8m をカウントしていないか（存在しない牌）
- テンパイ時の13枚→14枚完成手判定が正しく動作するか

### 12. 3人麻雀（サンマ）

- `num_players` による分岐が正しいか
- 壁残り枚数（55枚）
- チーなし、北抜きありの ActionConfig
- akagi_supreme3p の存在と auto-switch の正常動作

### 13. コード品質

- テストの充実度（追加すべきケースがないか）
- マジックナンバーの適切なコメント
- 自家副露の打点計算に `my_melds` が含まれているか（致命的バグ歴あり）
- 赤牌正規化の一貫性

## 重要な注意事項

> [!CAUTION]
> **Mortal の Q値は既に安全度・順位・打点・速度を内包している。**
> 外部から safety_weight や placement_adjustment を掛けると二重カウントになる。
> 介入は「Mortal が明確に苦手とする特定パターンの検出と上書き」に限定すること。

> [!WARNING]
> **`estimate_hand_value` や `count_dora_in_hand` では `my_hand` と `my_melds` の両方を参照すること。**
> 副露を無視すると打点が0になり、勝負手でオリるという致命的バグが過去に発生。

> [!IMPORTANT]
> **テストなしの変更は禁止。**
> 変更を加えたら必ず `pytest tests/test_akagi_supreme.py tests/test_akagi_eval8.py -v` を実行し、
> 必要に応じて新しいテストケースを追加すること。
